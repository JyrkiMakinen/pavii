Ejercicio análisis de un documento html con las herramientas del navegador Chrome.

Analice la página web del sitio www.frc.utn.edu.ar con Chrome devtools

Responda las siguientes preguntas:

¿Cuántos Kilobytes se transfieren al navegador para visualizar la página de manera completa? ¿Cuánto tiempo demora en cargar la página? Si se utiliza el caché del navegador, ¿demora lo mismo?

Se transfieren 131kB. Demora 11,14s en cargar sin caché, y 3,65s con caché.

¿La página posee errores o warnings?

Posee dos warnings y cuatro errores. Puedo deducir de los mensajes que se tratan de errores producto de mi configuración, ya que los cuatro warnings fueron de reproductores de publicidad bloqueados por el cliente, probablemente porque tengo instalado AdBlock en mi navegador.

¿Cuál es el archivo que demora más tiempo en descargarse de la página?

El que más demora es el archivo html llamado www.frc.utn.edu.ar, que es el que contiene la página a mostrar. Demora 1,77s con caché.

¿Todos los archivos provienen del mismo servidor?

No. Algunos provienen del servidor de la facultad, otros de YouTube, otros de Google, y otros sitios de publicidad.

¿Cuántas imágenes posee? ¿Qué tiempo que demora en descargarlas? ¿Cuál es la imagen de mayor tamaño en kilobytes que se descarga del sitio?

Hay 66 archivos de imagen, incluyendo formatos gif, jpeg, png y webp. Demoraron 424ms en descargarse con caché. La imagen de mayor tamaño fue h.gif de 1,4kB, aunque no se descargó por completo en la carga de la página porque había quedado guardada en la caché..

¿Cuántos archivos de hojas de estilo utiliza?

Hay 4 archivos de hojas de estilo, dos de los cuales son de YouTube.

¿En qué versión de html indica el documento que está realizada la página?

Está realizada en XHTML.

¿Los archivos javascript y/o css están minificados?

Ni los archivos javascript ni los css están minificados por completo.

¿Qué tiempo demoraría la página en descargar en un equipo con señal de datos de una red Regular 3G?

Tardaría 35,29s usando una red Slow 3G (antes llamada Regular 3G).

La página es compatible con el estándar html indicado en la cabecera. Analice la misma en https://validator.w3.org/

No fue posible validar la página ya que existen 23 errores en el código, principalmente relacionados con inconsistencias de apertura y cierre de etiquetas.

Analice las recomendaciones para mejorar la velocidad de carga de la página en https://developers.google.com/speed/pagespeed/

El rendimiento es de 66/100 para la versión de escritorio, y de 44/100 para la versión mobile. Las principales recomendaciones son reducir el tiempo de respuesta inicial del servidor para la carga del documento principal, y reducir código javascript fuera de uso. Esta última recomendación impacta aún más en la versión mobile, alcanzando los 9,42s de tiempo de carga innecesario.

Realice el mismo análisis con el sitio www.cba.gob.ar

El rendimiento es de 28/100 para la versión de escritorio, y de 17/100 para la versión mobile. Las principales recomendaciones son eliminar recursos de bloque renderizados y habilitar la compresión de texto. En la versión mobile, solucionar estos dos problemas eliminaría hasta 25,03s de tiempo de carga.
